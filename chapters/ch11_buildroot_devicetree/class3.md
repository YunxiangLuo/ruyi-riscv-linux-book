# Class 3: Device Tree、pinmux 与外设使能入门

## 本章目标

- 理解 Device Tree 描述硬件资源的作用。
- 认识 pinmux、外设节点和使能状态之间的关系。
- 明确内核、驱动和 BSP 的后续学习入口。

## 前置条件

- 已了解 Buildroot 产物和最小系统构建流程。
- 已完成 GPIO、UART、I2C/SPI 等用户态外设实验。

## 知识简介

Device Tree 是 Linux 识别板级硬件的重要入口。应用程序能否访问某个外设，不只取决于代码，还取决于内核配置、设备节点、pinmux 和板级使能状态。本节通过入门级观察建立应用程序、设备节点和板级配置之间的联系。

| 项目 | 要求 | 检查方式 |

| 构建环境 | 可选 | 能定位生成的 DTB |

## 操作步骤

1. 阅读一个 Device Tree 片段。
2. 找到 `compatible`、`status`、`pinctrl` 等关键属性。
3. 对照前序外设实验，说明用户态设备文件从何而来。
4. 观察 DTB 或系统启动日志中的外设信息。
5. 记录后续学习内核、驱动和 BSP 的入口。

## 常见问题

- 把 Device Tree 当作普通应用配置文件。
- 修改 DT 后没有重新构建、部署或重启。
- 过早深入驱动实现，导致偏离本课程主线。
