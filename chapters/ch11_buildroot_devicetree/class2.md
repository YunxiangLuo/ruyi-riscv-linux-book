# Class 2: 最小系统构建

## 本章目标

- 复现一个最小系统构建流程。
- 理解配置、下载、编译和输出目录之间的关系。
- 记录构建时间、依赖问题和输出产物。

## 前置条件

- 已完成 ch11 class1。
- 主机有足够磁盘空间和网络依赖准备。

## 知识简介

最小系统构建用于让学生看到嵌入式 Linux 系统从配置到产物的完整路径。本节聚焦可复现流程：选择配置，执行构建，定位输出，并说明镜像如何服务板级验证。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| Buildroot 环境 | 可构建或有课程实验包 | 能执行构建命令 |
| 主机磁盘 | 空间充足 | 检查剩余空间 |
| 依赖下载 | 可用或已缓存 | 记录下载状态 |

## 操作步骤

1. 选择课程提供的最小配置。
2. 执行配置命令。
3. 启动构建并记录开始时间。
4. 查看输出目录。
5. 记录镜像、rootfs、工具链等产物位置。
6. 汇总构建失败、下载失败或依赖缺失问题。

## 运行验证

| 验证项 | 预期现象 | 记录 |
| --- | --- | --- |
| 配置 | 能生成或加载最小配置 |  |
| 构建 | 能生成镜像或复现实验产物 |  |
| 产物 | 能指出输出目录和关键文件 |  |
| 记录 | 有构建时间和问题记录 |  |

## 常见问题

- 下载依赖失败时，应记录 URL、包名和错误信息。
- 构建时间较长时，应提前说明课堂可使用预构建产物。
- 输出目录混乱时，先从 Buildroot 标准 output 结构定位。
