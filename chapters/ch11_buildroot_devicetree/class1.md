# Class 1: Buildroot 产物识别

## 本章目标

- 理解 Buildroot 在嵌入式 Linux 系统构建中的定位。
- 认识工具链、根文件系统、内核镜像和启动镜像等常见产物。
- 区分系统构建和前序 Linux 应用开发的边界。

## 前置条件

- 已完成 Linux 应用开发主线。
- 了解基本交叉编译概念。

## 知识简介

Buildroot 属于系统扩展方向，用于帮助学生理解嵌入式 Linux 系统是如何构建出来的。它不改变全书应用开发主线，也不提前引入完整内核和驱动负担。本节课只要求学生识别产物和理解作用，为后续最小系统构建做准备。

## 操作步骤

1. 阅读 Buildroot 的定位说明。
2. 识别工具链、根文件系统、内核镜像和目标镜像。
3. 对照前序应用程序，说明它最终会进入哪个系统位置。
4. 记录哪些内容属于系统构建，哪些仍属于应用开发。
