# Class 1: 串口连接、TTY 工具与通信参数

## 本章目标

- 理解 UART 参数。
- 找到 Linux 串口设备。
- 使用串口工具连接开发板。

## 前置条件

- 已能启动开发板或目标系统。
- 已准备串口线或板载调试串口。
- 已阅读对应 `boards/` 目录中的板级说明。

## 知识简介

串口既是通信接口，也是板级调试生命线。即使网络未通，串口仍常用于查看启动日志、登录系统和输出调试信息。本节课重点掌握波特率、数据位、校验位、停止位等参数，以及 Linux 下 `/dev/tty*` 设备和串口工具的使用方法。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 串口线 | 连接正确 | 系统识别串口设备 |

| 串口参数 | 记录设备名、波特率、数据位、校验位、停止位 | 对照板级资料和实际连接 |

常用串口参数默认写法：

```text
baudrate: 115200
data bits: 8
parity: none
stop bits: 1
flow control: none
```

## 操作步骤

### 步骤 1：识别主机侧串口设备

插入 USB 转串口模块前后分别观察设备变化：

```bash
ls /dev/tty.*
ls /dev/cu.*
```

Linux 主机可使用：

```bash
ls /dev/ttyUSB*
ls /dev/ttyACM*
```

### 步骤 2：确认接线和参数

记录：

```text
板卡：
目标串口：
USB 转串口设备：
TX/RX 是否交叉：
GND 是否连接：
波特率：
```

LicheePi 4A 临时验证时，系统串口按 `UART0`、`U0-RX`、`U0-TX`、`115200` 准备。K1 / Muse Pi Pro 使用前必须补 `boards/k1/` 实测记录。

### 步骤 3：打开串口终端

任选一种工具：

```bash
screen /dev/ttyUSB0 115200
picocom -b 115200 /dev/ttyUSB0
minicom -D /dev/ttyUSB0 -b 115200
```

macOS 主机通常使用 `/dev/cu.*` 设备名，例如：

```bash
screen /dev/cu.usbserial-XXXX 115200
```

### 步骤 4：保存串口登录记录

上电或复位目标板，观察启动日志或登录提示。登录后执行：

```bash
hostname
whoami
uname -a
cat /etc/os-release
```

## 常见问题

- TX/RX 接反。
- 波特率不匹配。
- 串口设备权限不足。
- 只接了 TX/RX，没有接 GND。
- 使用了错误的主机设备名。
