# Lab 1: 串口连接、TTY 工具与通信参数

## 实验目标

- 打开串口设备。
- 实现串口输入回显。
- 记录串口设备名、波特率和连接方式。

## 实验环境

| 项目 | 要求 |
| --- | --- |
| 目标板 | K1 / Muse Pi Pro；K1 暂不可用时可用 LicheePi 4A 临时验证 |
| 串口连接 | USB 转串口模块或板载调试串口 |
| 串口参数 | 默认从 `115200 8N1` 开始，最终以板级记录为准 |
| 工具 | `screen`、`picocom`、`minicom` 或等价工具 |

LicheePi 4A 临时验证线索：系统串口是 `UART0`，连接 `U0-RX`、`U0-TX` 时需要 TX/RX 交叉并接 GND，波特率 `115200`。K1 / Muse Pi Pro 的最终串口接线和设备名需要补 `boards/k1/` 实测记录。

## 实验任务

### 任务 1：记录串口连接

```text
板卡：
目标串口：
主机设备名：
波特率：
数据位/校验位/停止位：
是否接 GND：
TX/RX 是否交叉：
登录方式：
```

### 任务 2：打开串口终端

任选一种工具：

```bash
screen /dev/ttyUSB0 115200
picocom -b 115200 /dev/ttyUSB0
minicom -D /dev/ttyUSB0 -b 115200
```

macOS 主机通常使用 `/dev/cu.*`：

```bash
screen /dev/cu.usbserial-XXXX 115200
```

### 任务 3：确认串口登录

登录后执行：

```bash
hostname
whoami
uname -a
cat /etc/os-release
```

### 任务 4：实现或运行 UART Echo

如果本课提供 Echo 程序，按 README 构建并运行；如果代码工程尚未提供，本实验先完成串口终端输入输出观察，并把 Echo 程序列为后续代码工程任务。

Echo 程序最小逻辑：

```text
打开串口设备
配置 115200 8N1
循环读取 1 个或多个字节
将收到的内容原样写回
```
