# Class 2: 工具链、Hello World、ELF 观察与 benchmark 边界

## 本章目标

- 理解 host、target、sysroot 的关系。
- 使用 C 编译器构建 Hello World。
- 在目标系统运行程序。
- 使用 `file`、`readelf` 和 `ldd` 观察二进制文件。
- 理解 benchmark 只作为环境对比和选做记录。

## 前置条件

- 已完成 ch02 class1 的工程工作区。
- 能进入 QEMU 或开发板系统。

## 知识简介

交叉编译的核心是区分“在哪里编译”和“在哪里运行”。host 是编译程序的主机，target 是程序最终运行的目标系统，sysroot 提供目标系统需要的头文件和库。Hello World 的意义不只是打印一行文字，而是建立“编译、传输、运行、观察 ELF 和依赖”的最小闭环。

benchmark 可以帮助记录不同环境的运行差异，但它不是本章主线能力。课程中只要求学生知道 benchmark 结果受 CPU、频率、镜像、负载、编译选项和测试方法影响，不能把一次简单结果当成开发板性能结论。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| C 编译器 | 主机编译器或交叉编译器可用 | `cc --version` 或交叉编译器版本 |
| 目标系统 | QEMU 或开发板 | 能进入 shell |
| 观察工具 | `file`、`readelf`、`ldd` | 命令可执行 |

## 操作步骤

1. 编写 Hello World 源码。
2. 使用主机编译器或课程指定工具链编译。
3. 将程序放到目标系统中运行。
4. 使用 `file` 观察文件类型和架构。
5. 使用 `readelf -h` 观察 ELF 头。
6. 使用 `ldd` 观察动态库依赖。
7. 可选执行一次课程指定 benchmark，记录命令、环境和结果。
8. 记录编译命令、运行输出和观察结果。

## 常见问题

- 编译器和目标架构不一致。
- 程序上传到了错误目录。
- 目标系统缺少运行依赖。
- `ldd` 对静态链接程序输出不同，需要在记录中说明。
