# Lab 1: Git 基础与工程工作区

## 实验目标

- 完成最小 Git 工作流记录。
- 建立源码、构建产物和日志分离的工程工作区。
- 学会用 `git status` 和 `git diff` 解释修改内容。

## 实验任务

1. 执行 `git --version` 和 `git status`。
2. 创建 `src/`、`build/`、`logs/` 目录。
3. 新建一份实验记录。
4. 执行 `git diff`，说明本次修改。
5. 记录哪些文件应提交，哪些文件属于构建产物或临时文件。

## 运行验证

| 验证项 | 预期现象 | 是否通过 |
| --- | --- | --- |
| Git 可用 | 输出 Git 版本 |  |
| 工作区状态 | 能说明当前修改 |  |
| 修改差异 | 能解释新增目录或记录文件 |  |
| 记录完整 | 有命令、输出和说明 |  |

## 提交要求

- Git 命令输出记录。
- 工程目录说明。
- `git status` 和 `git diff` 摘要。
