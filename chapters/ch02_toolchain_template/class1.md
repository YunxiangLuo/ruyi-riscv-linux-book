# Class 1: Git 基础与工程工作区

## 本章目标

- 掌握课程所需的最小 Git 工作流。
- 理解工作区、修改差异和实验记录之间的关系。
- 建立后续代码实验的文件组织习惯。

## 前置条件

- 已完成 ch01 环境闭环。
- 能在主机或目标系统中打开 shell。

## 知识简介

Git 在本课程中首先是实验记录工具。学生不需要一开始掌握复杂协作流程，但必须能查看当前工作区状态、识别自己改了哪些文件，并把源码、命令输出和调试记录组织在可追溯的位置。这样后续每个实验都能留下“做了什么、为什么失败、如何修复”的证据。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| Git | 已安装 | `git --version` |
| 工程目录 | 可写入 | `pwd`、`ls` |
| 实验记录 | 有记录目录 | `mkdir -p logs` |

## 操作步骤

1. 执行 `git --version`，确认 Git 可用。
2. 执行 `git status`，观察当前工作区状态。
3. 新建或修改一个实验记录文件。
4. 执行 `git diff`，观察修改内容。
5. 整理源码目录、构建目录和日志目录。
6. 在实验记录中写明本次修改目的。

| 目录组织 | 有源码、构建或日志目录 |  |

## 常见问题

- 把构建产物和实验日志混在一起，导致后续难以复查。
- 只保存成功输出，没有记录失败命令和错误信息。
- `git status` 中出现未预期文件时，先判断是否是构建产物、临时文件或实验记录。
