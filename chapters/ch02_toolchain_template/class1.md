# Class 1: 交叉编译模型与第一个程序

## 本章目标

- 理解 host、target、sysroot 的关系。
- 使用 GCC 编译并运行 Hello World。
- 使用 `readelf`、`ldd` 观察二进制文件。

## 前置条件

- 已完成 ch01 环境闭环。
- 能进入 QEMU 或开发板系统。

## 知识简介

交叉编译的核心是区分“在哪里编译”和“在哪里运行”。host 是编译程序的主机，target 是程序最终运行的目标系统，sysroot 则提供目标系统需要的头文件和库。学生需要先理解这三者的边界，再学习 GCC、ELF 和动态依赖观察。这样后续每个外设或 Linux 应用实验都可以复用同一套工程思路。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 工具链 | RISC-V GCC 或课程指定工具链 | `gcc --version` 或交叉编译器版本 |
| 目标系统 | QEMU 或开发板 | 能运行测试程序 |

## 操作步骤

1. 编写 Hello World。
2. 使用 GCC 编译。
3. 在目标系统运行。
4. 使用 `readelf` 和 `ldd` 观察结果。

## 常见问题

- 编译器和目标架构不一致。
- 程序上传到了错误目录。
- 目标系统缺少运行依赖。
