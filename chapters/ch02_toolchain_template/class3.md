# Class 3: Makefile 控制编译与工程模板

## 本章目标

- 建立后续章节可复用的 C 工程模板。
- 使用 Makefile 管理构建、运行和清理。
- 记录一次构建失败、排查和修复过程。

## 前置条件

- 已完成 ch02 class1 和 class2。
- 已能编译并运行 Hello World。

## 知识简介

课程后续实验会不断增加源码、头文件、配置、日志和数据目录。统一工程模板可以减少重复组织成本。调试记录从本课开始进入实验常规要求：不仅提交成功输出，也要保留失败命令、错误信息、判断依据和修复结果。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| Make | 可用 | `make --version` |
| 编译器 | 可用 | `cc --version` 或交叉编译器版本 |
| Git | 可用 | `git status` |

## 操作步骤

### 步骤 1：创建工程目录

```bash
mkdir -p src include build logs
```

### 步骤 2：编写 Makefile

至少包含 `build`、`run`、`clean`。

### 步骤 3：制造并修复一次错误

记录失败输出和修复过程。

## 运行验证

| 验证项 | 预期现象 | 记录 |
| --- | --- | --- |
| `make build` | 构建成功 |  |
| `make run` | 程序运行 |  |
| `make clean` | 清理构建产物 |  |
| 调试记录 | 有失败输出和修复说明 |  |

## 常见问题

### Makefile 路径错误

现象：找不到源文件或目标文件。

处理：检查变量、目录和缩进。
