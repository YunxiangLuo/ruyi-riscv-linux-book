# Class 2: 轮询、消抖与状态机式交互

## 本章目标

- 理解按键抖动。
- 实现软件消抖。
- 用状态机组织交互逻辑。

## 前置条件

- 已完成 GPIO 输入和输出。

## 知识简介

按键输入不是理想的瞬时信号，机械抖动会导致一次按键被程序识别为多次触发。轮询和消抖可以让输入更稳定，而状态机可以把按键事件和 LED 模式切换组织成清晰逻辑。通过这一节，学生会从“能控制外设”进入“能设计交互”的阶段。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| GPIO 输入 | 按键可读 | 运行 Button |
| GPIO 输出 | LED 可控 | 运行 Blink |

## 操作步骤

1. 观察未消抖按键效果。
2. 加入延时或稳定计数消抖。
3. 设计 LED 模式状态机。

## 运行验证

| 验证项 | 预期现象 | 记录 |
| --- | --- | --- |
| 消抖 | 单次按键不会重复触发 |  |
| 状态机 | LED 模式按规则切换 |  |

## 常见问题

- 消抖时间太短。
- 状态转移条件不完整。
