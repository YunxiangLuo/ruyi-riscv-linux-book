# Class 2: 任务拆分与生产者消费者

## 本章目标

- 将程序拆分成多个任务。
- 使用队列组织生产者消费者模型。
- 理解 Mailbox 的选学边界。

## 前置条件

- 已能创建线程并使用互斥。

## 知识简介

采集、显示、记录如果都写在一个循环中，程序会很快变得难以维护。任务拆分能让每个线程职责清晰，生产者消费者模型能让数据在任务之间稳定流动。Mailbox 作为扩展概念，只需要理解用途和边界，不作为主线实现负担。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 线程程序 | 可运行 | 线程日志 |
| 队列结构 | 可读写 | 数据流测试 |

## 操作步骤

1. 划分采集和显示任务。
2. 建立队列。
3. 实现生产者和消费者。
4. 讨论 Mailbox 边界。
