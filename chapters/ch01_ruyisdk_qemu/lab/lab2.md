# Lab 2: 启动 QEMU 或开发板并远程登录

## 实验目标

- 启动 QEMU `virt` 或真实开发板。
- 完成 SSH/SCP/串口登录路径验证。

## 实验环境

| 项目 | 要求 |
| --- | --- |
| 目标环境 | QEMU `virt` 或 RISC-V Linux 开发板 |
| 登录工具 | SSH、SCP 或串口工具 |
| 软件依赖 | 按课程环境准备 |

## 实验任务

1. 启动目标系统。
2. 登录目标系统。
3. 向目标系统传输一个测试文件。
4. 记录启动日志和登录方式。

## 运行验证

| 验证项 | 预期现象 | 是否通过 |
| --- | --- | --- |
| 系统启动 | 能进入 Linux shell |  |
| 远程登录 | SSH 或串口登录成功 |  |
| 文件传输 | 测试文件出现在目标系统 |  |

## 提交要求

- 启动日志摘要。
- 登录命令和结果。
- 文件传输命令和结果。
