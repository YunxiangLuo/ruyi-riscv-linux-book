# Class 3: 日志、错误处理与可维护目录结构

## 本章目标

- 输出可诊断的运行日志。
- 使用返回值和错误信息定位问题。
- 整理应用程序目录结构。

## 前置条件

- 已完成文件 I/O 和配置文件实验。
- 已有一个可运行示例程序。

## 知识简介

能运行的程序还不等于可维护的程序。日志、错误处理和目录结构让程序在后续项目中可诊断、可复现、可部署。本节把这些能力作为 ch08 综合项目的直接基础。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 日志目录 | 可写 | `mkdir -p logs` |
| 示例程序 | 可运行 | `make run` |
| 配置文件 | 可读取 | `cat config/app.conf` |

## 操作步骤

### 步骤 1：加入日志

记录启动、配置加载、数据写入和错误。

### 步骤 2：构造错误

故意使用不存在路径或错误权限。

### 步骤 3：整理目录

固定 `config/`、`data/`、`logs/` 和 `build/`。

## 常见问题

### 日志为空

现象：程序运行但没有日志。

处理：检查路径、权限和刷新时机。
