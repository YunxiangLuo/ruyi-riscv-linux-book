# Lab 3: 日志、错误处理与可维护目录结构

## 实验目标

- 写入运行日志。
- 构造并记录一个错误。
- 整理应用程序目录结构。

## 实验环境

| 项目 | 要求 |
| --- | --- |
| 主机环境 | 可构建示例程序 |
| 目标环境 | QEMU 或开发板 |
| 软件依赖 | Makefile 工程 |

## 实验任务

### 任务 1：写日志

记录程序启动、配置读取、数据写入。

### 任务 2：构造错误

使用错误路径或权限，记录错误输出。

### 任务 3：整理目录

形成 `config/`、`data/`、`logs/`、`build/`。

## 实验步骤

```bash
make build
make run
find . -maxdepth 2 -type d
```

## 运行验证

| 验证项 | 预期现象 | 是否通过 |
| --- | --- | --- |
| 日志 | 文件中有运行事件 |  |
| 错误 | 有可诊断错误信息 |  |
| 目录 | 结构符合模板 |  |

## 常见问题

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 无法写日志 | 目录不存在或无权限 | 创建目录并检查权限 |

## 提交要求

- 日志文件。
- 错误复现和修复记录。
- 目录结构。
