# Class 2: 日志、错误处理与程序目录结构

## 本章目标

- 写入运行日志。
- 处理常见错误。
- 整理可维护程序目录。

## 前置条件

- 已完成文件 I/O 和配置文件。

## 知识简介

日志和错误处理让程序变得可诊断。程序目录结构让源码、配置、日志和数据各有位置，便于后续线程和综合项目复用。本节课重点是让学生从“程序能跑”升级到“程序出了问题能查”。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 日志目录 | 可写 | 生成日志文件 |
| 配置目录 | 可读 | 加载配置 |

## 操作步骤

1. 增加日志模块。
2. 增加错误处理。
3. 整理目录结构。
