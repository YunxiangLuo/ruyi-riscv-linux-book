# Class 4: HTTP 状态页发布与综合验收

## 本章目标

- 发布 HTTP 状态页展示最新数据。
- 完成联网信息采集终端综合验收。
- 整理演示、日志和交付材料。

## 前置条件

- 已完成项目骨架、采集显示任务和日志配置。
- 目标系统网络可用。

## 知识简介

HTTP 状态页是本课程主线项目的默认网络发布方式。它足够直观，便于在浏览器中验收，也避免把 MQTT、云平台或复杂 Web 框架提前放入必做主线。综合验收关注项目能否连续运行、可观察、可解释、可复现。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 网络 | 目标板有 IP | `ip addr` |
| HTTP 服务 | 可启动 | 访问状态页 |
| 日志 | 可追踪运行状态 | `logs/` |

## 操作步骤

### 步骤 1：启动项目

运行综合项目程序。

### 步骤 2：访问状态页

在浏览器或 `curl` 中访问目标板 HTTP 地址。

### 步骤 3：完成综合验收

记录数据、日志、截图和失败排查。

## 常见问题

### 页面无法访问

现象：浏览器连接失败。

处理：检查目标 IP、端口、防火墙和服务日志。
