# Class 3: 日志、配置与运行状态管理

## 本章目标

- 默认实现 HTTP 状态页。
- 将采集数据发布到局域网。
- 明确 MQTT/Socket 是拓展路径。

## 前置条件

- 已完成项目本地运行闭环。
- 目标系统具备网络连接。

## 知识简介

本课程的综合项目默认采用 HTTP 状态页作为联网发布方式。这样学生可以通过浏览器直接访问开发板状态，验收清晰、依赖较少。MQTT 和 Socket 可作为拓展，不作为全体学生默认负担。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 网络 | 主机能访问开发板 | ping 或浏览器访问 |
| HTTP 服务 | 可启动 | 端口监听 |

## 操作步骤

1. 设计状态数据格式。
2. 实现 HTTP 状态页。
3. 展示最新采集数据。
4. 保留 MQTT/Socket 拓展接口说明。
