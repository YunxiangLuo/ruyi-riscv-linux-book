# Class 3: MFCC/int8 推理部署

## 本章目标

- 理解 MFCC 特征到 int8 推理输入的连接关系。
- 在板端构建并运行最小 KWS 推理程序。
- 记录推理耗时、置信度和低置信度回退策略。

## 前置条件

- 已准备语音样本或预录音频。
- 已建立 TinyML KWS 工程目录。
- 已具备 Makefile 构建和日志记录能力。

## 知识简介

MFCC 用于把音频转换为适合轻量模型处理的特征，int8 推理用于降低模型运行时的计算和内存压力。本节课重点不是训练复杂模型，而是让学生看清板端部署链路：输入音频或特征，生成固定维度张量，调用推理函数，输出关键词类别、置信度和运行耗时。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 特征文件 | 维度固定、可读取 | 能打印特征长度 |
| 模型参数 | 可被工程包含 | `make build` 可通过 |
| 运行日志 | 可记录推理结果 | 日志包含关键词和耗时 |

## 操作步骤

1. 检查 MFCC 特征维度和模型输入维度是否一致。
2. 将 int8 模型参数或 `weights.h` 加入工程。
3. 编写最小推理入口，输出关键词、置信度和耗时。
4. 使用预录音频或固定特征文件进行重复测试。
5. 记录低置信度、未知词和噪声输入的处理方式。

## 常见问题

- 特征维度不一致时，先固定采样率、窗口长度和帧数。
- 模型参数包含失败时，检查头文件路径和类型定义。
- 结果一直偏向同一类时，先用固定样本排除输入链路问题。
