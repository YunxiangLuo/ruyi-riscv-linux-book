# Class 2: 板上 KWS 推理与 LED 控灯集成

## 本章目标

- 在 RISC-V Linux 板上运行 int8 KWS 推理程序。
- 将“开/关/闪”关键词映射为 LED 动作。
- 记录关键词、置信度、耗时和失败回退。

## 前置条件

- 已完成语音样本准备。
- 已导出 MFCC 特征或 `weights.h`。
- 已具备可复用 LED 控制代码。

## 知识简介

本节课把 KWS 项目从“能推理”推进到“能控制设备”。程序需要读取音频或特征输入，执行 int8 推理，输出关键词和置信度，再通过状态机控制 LED：识别“开”时点亮，识别“关”时熄灭，识别“闪”时进入闪烁模式。若实时音频链路不稳定，可以使用预录音频文件作为等效输入，但最终仍需形成可演示的板上推理和控灯闭环。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| KWS 推理程序 | 可构建、可运行 | 输出关键词和置信度 |
| LED 设备 | 可控制 | 能亮灭或闪烁 |
| 日志记录 | 可写入 | 记录推理结果和耗时 |

## 操作步骤

1. 编译板上 KWS 推理程序。
2. 使用预录音频或实时音频触发推理。
3. 输出关键词、置信度和推理耗时。
4. 编写关键词到 LED 动作的映射。
5. 连续测试“开/关/闪”三类指令。
6. 记录误识别、低置信度和回退策略。

## 常见问题

- 关键词置信度低时，先检查样本质量和背景噪声。
- LED 无响应时，先单独运行 ch03 LED 控制程序。
- 实时音频不稳定时，先用预录音频验证推理链路。
