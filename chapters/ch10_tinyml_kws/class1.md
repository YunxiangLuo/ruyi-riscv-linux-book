# Class 1: KWS 控灯最小流程

## 本章目标

- 理解 TinyML KWS 控灯项目的输入、推理和输出链路。
- 建立“音频或预录样本 -> 关键词结果 -> LED 动作”的最小演示路径。
- 明确实时音频、模型训练和板端部署的风险边界。

## 前置条件

- 已完成 GPIO LED 控制。
- 已完成 Makefile 工程模板。
- 已具备日志记录和基本线程协作能力。

## 知识简介

TinyML KWS 语音关键词识别控灯把音频输入、特征提取、轻量模型推理和板级输出连接成一个完整项目。本节课先建立最小闭环，不把难点一次性堆满：可以先用预录音频或模拟识别结果触发 LED，确认项目的数据流、接口和验收方式，再进入样本准备、MFCC/int8 推理和真实控灯集成。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| LED 控制 | 可复用 ch03 模块 | LED 能亮灭 |
| 工程模板 | 可构建 | `make build` 可执行 |
| 音频输入 | 麦克风或预录音频可选 | 能列出样本或说明模拟输入 |

## 操作步骤

1. 建立 KWS 项目目录，准备 `audio/`、`model/`、`src/`、`logs/`。
2. 画出音频输入、特征、推理、关键词和 LED 动作之间的数据流。
3. 使用模拟关键词或预录音频触发一次 LED 动作。
4. 记录当前哪些环节是真实运行，哪些环节仍是占位或待验证。
5. 制定后续样本、推理和集成的验收标准。

## 常见问题

- 一开始就要求实时语音识别，导致硬件和模型问题叠加。
- 没有记录模拟输入和真实输入的差异。
- LED 控制失败时，应回到 ch03 单独验证。
