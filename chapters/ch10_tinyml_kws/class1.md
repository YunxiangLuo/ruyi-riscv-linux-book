# Class 1: TinyML KWS 流程、样本与特征

## 本章目标

- 跑通 TinyML KWS 控灯的最小流程。
- 准备“开/关/闪”语音样本或预录音频。
- 理解 MFCC 特征和 int8 量化导出的作用。

## 前置条件

- 已完成 GPIO LED 控制。
- 已完成 Makefile 工程模板。
- 已具备日志记录和基本线程协作能力。

## 知识简介

TinyML KWS 语音关键词识别控灯把音频输入、特征提取、轻量模型推理和板级输出连接成一个完整项目。学生可以使用麦克风实时采集，也可以先用预录音频降低硬件风险。本节课重点建立“音频文件或麦克风 -> MFCC 特征 -> int8 模型参数 -> 关键词结果”的数据链路，并明确最终要把“开/关/闪”映射到 LED 动作。

## 环境准备

| 项目 | 要求 | 检查方式 |
| --- | --- | --- |
| 音频输入 | 麦克风可用或预录音频可读取 | 能列出或播放样本 |
| 工程模板 | 可构建 | `make build` 可执行 |
| LED 控制 | 可复用 ch03 模块 | LED 能亮灭 |

## 操作步骤

1. 建立 KWS 项目目录，准备 `audio/`、`model/`、`src/`、`logs/`。
2. 采集或导入“开/关/闪”三类语音样本。
3. 记录样本命名规则、采样率和时长。
4. 运行 MFCC 特征提取流程。
5. 导出 int8 模型参数或 `weights.h`。

## 常见问题

- 麦克风不可用时，先使用预录音频注入。
- 样本过短或噪声过大时，应重新采集并记录环境。
- 量化导出失败时，先检查模型输入维度和特征维度是否一致。
