# TinyML KWS 语音关键词识别控灯项目定义

## 项目定位

本项目作为 ch10 默认扩展项目，目标是在 RISC-V Linux 开发板上完成“音频输入 -> MFCC 特征 -> int8 KWS 推理 -> 关键词 -> LED 动作”的完整闭环。Face Detection、YOLO 和大模型控制设备只作为备选拓展，不作为 ch10 默认验收路径。

## 复用章节

| 来源章节 | 复用能力 | 在本项目中的用途 |
| --- | --- | --- |
| ch01 | RuyiSDK、QEMU/板卡登录、SCP/串口 | 部署程序、上传样本、查看日志 |
| ch02 | Makefile、工程目录、ELF 和依赖观察 | 构建 KWS 推理程序 |
| ch03 | GPIO LED 控制、状态机 | 执行“开/关/闪”动作 |
| ch04 | 串口日志和命令回显 | 输出关键词、置信度和调试信息 |
| ch06 | 文件、配置、日志 | 管理样本、配置阈值、记录推理结果 |
| ch07 | 线程和生产者-消费者 | 支撑音频采集与推理任务协同 |

## 默认实现

| 项目 | 默认要求 |
| --- | --- |
| 关键词 | “开”“关”“闪” |
| 输入 | 麦克风实时输入；若硬件不稳定，允许预录音频文件注入 |
| 特征 | MFCC |
| 推理 | int8 KWS 轻量模型 |
| 输出 | LED 点亮、熄灭、闪烁 |
| 日志 | 关键词、置信度、推理耗时、失败回退 |

## 实验拆分

| 实验 | 任务 | 验收 |
| --- | --- | --- |
| Lab 1 | 准备“开/关/闪”样本，提取 MFCC，导出 int8 参数或 `weights.h` | 能生成样本清单、特征结果和模型参数 |
| Lab 2 | 部署板上 KWS 推理程序，集成 LED 控制 | 喊“开”灯亮，喊“关”灯灭，喊“闪”LED 闪烁 |

## 备用方案

| 风险 | 备用处理 |
| --- | --- |
| 麦克风不可用 | 使用预录音频文件作为输入 |
| 实时音频链路不稳定 | 先用文件注入完成推理和控灯闭环 |
| KWS 模型准备超出课时 | 使用预导出的 `weights.h`，课堂重点放在板上推理和动作映射 |
| 语音方案整体受阻 | 使用温湿度异常检测 fallback，但仍保留 TinyML 训练、量化、部署和 LED 告警闭环 |

## 验收标准

- 能在目标板或等效实验环境中运行 KWS 推理程序。
- 能输出关键词、置信度和推理耗时。
- 喊“开”时 LED 点亮。
- 喊“关”时 LED 熄灭。
- 喊“闪”时 LED 闪烁。
- 能说明预录音频 fallback 与实时麦克风方案的边界。
