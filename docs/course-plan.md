# 课程计划表

本计划表由 `docs/course-outline.md` 中已经定好的课程大纲出发，并参考 RuyiSDK 资料、教材评审意见、开发板课程案例和嵌入式 Linux 应用开发资料整理而来。

## 使用原则

- 已定稿的课程大纲不在本计划表中改动；本计划表只负责把大纲拆成可执行的开发计划。
- ch01-ch07 的实验必须成为 ch08 综合项目的基础步骤，不能设计成彼此孤立的一次性实验。
- ch08 只做前七章能力整合、项目接口对齐、HTTP 状态页和综合验收，不重复讲授 GPIO、OLED、文件 I/O、pthread 等基础内容。
- ch09-ch11 每章都必须形成一个完整扩展项目，不能只停留在知识点介绍。
- ch10 默认项目调整为 TinyML KWS 语音关键词识别控灯，关键词为“开/关/闪”，LED 作为验收输出。
- Face Detection / YOLO 不再作为 ch10 默认最终作品，只作为后续备选或拓展。
- 大模型 / Agent / Workflow 等新增内容先进入扩展项目池，不直接改动已定主线大纲。
- 授课章节应提交：课程文档、授课 PPT。
- 实验章节应提交：实验指导书。
- 完成后在对应交付物列标记状态。
- 建议状态：`未开始`、`进行中`、`已完成`、`待审核`、`需修改`、`终审通过`。
- `计划完成时间`、`实际提交时间`、`审核日期`、`责任人` 可根据项目管理安排补充。

## 主线项目依赖链

ch01-ch07 的实验不是孤立练习，必须成为 ch08 “联网信息采集终端”的基础步骤。ch08 只做整合、接口对齐、线程架构升级、HTTP 状态页和综合验收。

| 前置章节 | 前置实验能力 | ch08 复用位置 | 计划要求 |
| --- | --- | --- | --- |
| 第 1 章 | 安装 `ruyi`、启动 QEMU/板卡、SSH/SCP/串口登录 | 项目部署、远程运行、日志拉取、现场演示 | 实验必须保留远程登录和文件传输 |

| 第 3 章 | Blink、Button、消抖、状态机 | 状态指示、告警灯、按键切换模式或手动触发 | GPIO 实验应沉淀为可复用模块 |

| 第 5 章 | PWM、ADC、I2C/SPI、OLED、DHT22、结果可视化 | 环境数据采集、本地显示、告警输出 | 采集与显示实验形成传感终端雏形 |
