# ch08 主线项目：联网信息采集终端

## 项目定位

本项目是课程主线作品，用于整合 ch01-ch07 的环境、工程、板级交互、外设采集、文件日志和多线程能力。ch08 不重新讲授 GPIO、OLED、文件 I/O、pthread 等基础内容，只做模块组装、接口对齐、项目级线程架构、HTTP 状态页和综合验收。

## 项目目标

- 复用前七章实验产物，形成一个可持续运行的 Linux 应用。
- 周期性采集环境数据或 fallback 模拟数据。
- 在本地显示当前状态，并通过 LED 表示运行或告警状态。
- 将运行状态、采集数据和日志摘要发布为 HTTP 状态页。
- 通过浏览器、串口或日志完成可观察验收。

## 功能模块

| 模块 | 职责 | 输入 | 输出 |
| --- | --- | --- | --- |
| 配置模块 | 读取采样周期、HTTP 端口、告警阈值和日志级别 | `config/app.conf` | 运行参数 |
| 采集模块 | 周期性读取 DHT22、ADC 或模拟数据 | 传感器或模拟数据源 | 环境数据结构 |
| 状态模块 | 维护最新数据、运行状态和告警状态 | 采集数据、错误事件 | 共享状态 |
| 显示模块 | 刷新 OLED 或终端状态 | 共享状态 | 本地状态页面 |
| LED 模块 | 输出心跳、告警或手动模式 | 运行状态、告警状态、按键事件 | LED 动作 |
| 日志模块 | 写入运行日志、数据日志和错误日志 | 状态变化、采集数据、错误事件 | `logs/`、`data/` |
| HTTP 模块 | 对外发布状态页 | 共享状态、日志摘要 | 浏览器可访问页面 |
| 串口命令模块 | 提供调试查询入口 | 串口输入 | `status`、`config`、`help` 响应 |

## 线程结构

```text
main
├── config load
├── sensor_thread       # 采集 DHT22/ADC/模拟数据
├── display_thread      # 刷新 OLED 或终端状态
├── log_thread          # 写入数据日志和运行日志
├── http_thread         # 发布 HTTP 状态页
└── uart_command_thread # 可选：串口 status/config/help
```

线程间共享一份 `system_state`，至少包含最新传感数据、运行时间、告警状态、最近错误、网络状态和最近更新时间。共享状态必须通过互斥锁或消息队列保护。

## 数据结构建议

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `temperature` | float | 温度，单位摄氏度；模拟数据可填测试值 |
| `humidity` | float | 湿度，单位百分比；无 DHT22 时可为空或模拟 |
| `adc_value` | int | ADC 原始值或归一化值 |
| `sample_time` | string | 最近采样时间 |
| `uptime_sec` | uint64 | 程序运行秒数 |
| `alarm` | bool | 是否超过阈值 |
| `status` | string | `OK`、`WARN`、`ERROR` |
| `last_error` | string | 最近错误摘要 |

## HTTP 状态页

HTTP 状态页是 ch08 的默认新增能力，不要求学生深入实现完整 HTTP 框架。可以使用最小 HTTP server、板卡已有示例或课程提供的轻量模板。

状态页至少显示：

- 项目名称：联网信息采集终端
- 运行状态：`OK` / `WARN` / `ERROR`
- 最新温度、湿度、ADC 或模拟数据
- 最近更新时间
- 程序运行时长
- 告警状态
- 最近错误摘要
- 日志文件位置或最近日志摘要

验收 URL 建议：

```text
http://<board-ip>:8080/
```

## 验收标准

- 项目能在 K1 / Muse Pi Pro、LicheePi 4A 临时验证环境或指定目标环境中构建和运行。
- 程序能持续运行至少 10 分钟，不出现明显崩溃或线程卡死。
- 能周期性产生环境数据；真实传感器不可用时，模拟数据 fallback 可用。
- OLED 或终端能显示最新数据和运行状态。
- LED 能表示运行心跳或告警状态。
- `logs/` 和 `data/` 中能看到可读日志或数据记录。
- 浏览器访问 HTTP 状态页能看到最新数据、运行状态和最近更新时间。
- 串口或终端能输出关键运行日志。

## 不做内容

- 不把 Socket、HTTP 协议细节扩展成独立理论重点。
- 不强制所有学生实现 MQTT、云平台或复杂 Web UI。
- 不在 ch08 重新讲授 GPIO、OLED、DHT22、文件 I/O、pthread 基础。
- 不绑定多开发板适配；第一轮以 K1 / Muse Pi Pro 目标板口径和 LicheePi 4A 临时验证环境为主。

## 风险与 fallback

| 风险 | 影响 | fallback |
| --- | --- | --- |
| DHT22 不稳定或缺货 | 无法采集真实温湿度 | 使用 ADC 或模拟数据 |
| OLED 适配困难 | 本地显示不可见 | 使用终端状态输出 |
| 网络配置复杂 | HTTP 页面无法访问 | 先在目标机本地 `curl 127.0.0.1:8080` 验证，再处理局域网访问 |
| 多线程竞争 | 数据显示和日志异常 | 降级为采集线程 + HTTP 线程，日志同步写入 |
| 串口资源不足 | 命令回显不可用 | 使用 SSH 终端命令或日志查询 |
